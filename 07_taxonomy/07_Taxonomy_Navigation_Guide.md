## How to Run the Script

From the repository root directory:

```
python notebooks/stage4_output.py
```

No additional arguments are needed. The script reads from `02_openings/parsed_openings.parquet` and `07_taxonomy/final_classes.parquet`, then writes all outputs into `06_clusters/` and `07_taxonomy/`.

To install the required dependencies:

```
pip install -r requirements.txt
pip install scipy
```

## Output Files — Tables (CSV)

These are the quantitative results. All CSV files open in Excel, Google Sheets, or any text editor.


### 1. `07_taxonomy/cluster_statistics.csv`

**What it contains:** One row for the total sample and one row for each of the 8 clusters. Columns include the number of reviews, percentage of the total, mean and median word count, mean and median star rating, mean, median, standard deviation, and maximum of helpfulness votes.

**How to read it:** The first row ("TOTAL SAMPLE") gives the baseline. Each subsequent row shows one cluster. Compare across rows to see which clusters have longer reviews, higher ratings, or more helpfulness votes.

| Column | Meaning |
|--------|---------|
| `cluster_id` | The numeric cluster label (0–7), or "ALL" for the total |
| `cluster_name` | The human-readable class name (e.g., "Affective Stance") |
| `n_reviews` | How many reviews belong to this cluster |
| `pct_of_total` | That count as a percentage of all 9,999 reviews |
| `mean_word_count` | Average review length in words |
| `mean_star_rating` | Average star rating (1.0–5.0) |
| `mean_helpful_votes` | Average number of helpfulness votes received |


### 2. `07_taxonomy/cluster_sample_reviews.csv`

**What it contains:** 10 randomly sampled reviews from each cluster (80 reviews total). Each row includes the cluster label, the review's opening sentence, the full review text, star rating, helpfulness votes, word count, and product domain.

**How to read it:** This file provides qualitative evidence — it lets you read actual reviews from each cluster and verify that the cluster labels make intuitive sense. For example, reviews in the "Terse Evaluation" cluster should appear noticeably shorter and more fragment-like than those in "Contextual Framing."

A markdown-formatted version of this same data is also saved as `cluster_sample_reviews.md` for easier reading on GitHub.


### 3. `07_taxonomy/cluster_example_openings.csv`

**What it contains:** 5 real opening sentences per cluster, displayed alongside the star rating and product domain. This is a focused version of the sample reviews file that highlights only the opening — the first words of the review that the clustering was based on.

**How to read it:** This is the most direct way to understand what each cluster "looks like." Compare openings across clusters:

| Cluster | Example Opening |
|---------|----------------|
| Product Reference | "This book is a great introduction to..." |
| Affective Stance | "I have to agree with many readers that..." |
| Terse Evaluation | "Books 1,2 3 were really good." |
| Specific Identification | "At the end of 'A Feast for Crows' (AFFC) we..." |


### 4. `07_taxonomy/cluster_domain_crosstab.csv`

**What it contains:** A cross-tabulation showing what percentage of each cluster comes from the Books domain versus the Electronics domain. Each row is one cluster, each column is one domain, and values are percentages that sum to 100% across each row.

**How to read it:** If a cluster shows 70% Books and 30% Electronics, that opening style is disproportionately used in book reviews. If both domains are close to 50/50, the opening style generalises across product categories.


### 5. `07_taxonomy/kruskal_wallis_tests.csv`

**What it contains:** Results of the Kruskal-Wallis H-test, a non-parametric statistical test that checks whether the 8 clusters differ significantly on three metrics: star rating, word count, and helpfulness votes.

**How to read it:** Each row tests one metric. The key columns are:

| Column | Meaning |
|--------|---------|
| `H_statistic` | The test statistic — larger values mean bigger differences between clusters |
| `p_value` | The probability that the observed differences are due to chance |
| `significance` | A shorthand: `***` means p < .001 (very significant), `**` means p < .01, `*` means p < .05, `n.s.` means not significant |

If all three metrics show `***`, you can state in the thesis that the clusters differ significantly on every measured dimension. This is the Kruskal-Wallis test rather than ANOVA because star ratings and helpfulness votes are not normally distributed, and Kruskal-Wallis does not require that assumption.

## Output Files — Figures (PNG)

These are the visual comparisons. All are saved as high-resolution PNG files (200 DPI) suitable for inclusion in the thesis document.


### Figure 1. `06_clusters/cluster_distributions_overlay.png`

**What it shows:** Three histograms (star rating, word count, helpfulness votes) with all 8 clusters overlaid on the same axes using different colors. This gives a quick sense of how much the distributions overlap.

**When to use it:** When you want to show the overall picture — are the clusters mostly overlapping or clearly separated?


### Figure 2. `06_clusters/cluster_distributions_by_class.png`

**What it shows:** The same three metrics, but each cluster gets its own small subplot (known as "small multiples"). This avoids the visual clutter of overlaying 8 distributions.

**When to use it:** When the overlay plot is too crowded and you want to compare individual cluster shapes side by side.


### Figure 3. `06_clusters/cluster_kde_curves.png`

**What it shows:** Smooth density curves (kernel density estimation) for each cluster, one line per cluster, across three panels: star rating, word count, and helpfulness votes. Unlike histograms, KDE curves show the shape of each distribution as a continuous line.

**When to use it:** This is the cleanest comparison figure. Where two curves peak at different locations, the clusters genuinely differ on that metric. Where curves overlap heavily, the clusters are similar.


### Figure 4. `06_clusters/cluster_radar_comparison.png`

**What it shows:** A radar (spider) chart where each cluster is drawn as a polygon with three axes: mean star rating, mean word count, and mean helpful votes. Values are normalised to a 0–1 scale so the three metrics are visually comparable.

**When to use it:** To show each cluster's "profile" at a glance. A cluster with a large polygon is high on all metrics. A cluster that extends far on one axis but not others has a distinctive signature.


### Figure 5. `06_clusters/cluster_means_comparison.png`

**What it shows:** Three grouped bar charts showing the mean star rating, mean word count, and mean helpfulness votes for each cluster. Each bar is labeled with its exact value.

**When to use it:** When you need precise numeric comparisons rather than distribution shapes.


### Figure 6. `06_clusters/cluster_domain_heatmap.png`

**What it shows:** A color-coded grid where rows are clusters and columns are product domains (Books, Electronics). Each cell shows the percentage of that cluster's reviews that come from that domain. Darker red cells indicate higher concentration.

**When to use it:** To answer: "Do certain opening styles appear more in one domain than the other?" This directly supports the cross-domain analysis in the thesis (RQ4).


### Figure 7. `06_clusters/cluster_box_plots.png`

**What it shows:** Box-and-whisker plots for each metric, grouped by cluster. Each box shows the median (center line), the interquartile range (the box, covering the 25th to 75th percentile), and outliers (individual dots beyond the whiskers).

**When to use it:** When you want to show not just averages but the spread and variability within each cluster. Two clusters can have similar means but very different spreads — box plots reveal this while bar charts do not.

## How the Script Works — Technical Summary

The script (`notebooks/stage4_output.py`) performs the following steps in order:

1. **Loads** the review corpus (9,999 reviews with text, ratings, votes, and syntactic parse data) and the taxonomy file (the same reviews with cluster assignments from Stage 4 clustering).

2. **Merges** the two files on `reviewerID` so each review has both its original features and its cluster label.

3. **Generates sample reviews** — randomly selects 10 reviews per cluster and saves them as both CSV and markdown.

4. **Computes statistics** — calculates mean, median, and standard deviation of word count, star rating, and helpfulness votes for the total sample and for each cluster.

5. **Generates distribution plots** — histograms overlaid and as small multiples.

6. **Generates comparison curves** — KDE density curves, radar chart, and grouped bar chart.

7. **Generates the domain heatmap** — cross-tabulates cluster membership by product domain.

8. **Generates box plots** — shows spread and outliers for each metric by cluster.

9. **Generates example openings** — saves 5 representative opening sentences per cluster.

10. **Runs statistical tests** — Kruskal-Wallis H-test for each metric to confirm that cluster differences are statistically significant.
